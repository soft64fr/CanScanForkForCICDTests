Comprehensive input data structure for QR code generation and rendering.

Encapsulates all user-provided data and configuration parameters required to generate, style, and display a QR code. Supports both MECARD (structured contact) and FREE (arbitrary text) encoding modes.

This record serves as an immutable data transfer object (DTO) that aggregates:

  • Contact information (MECARD mode)
  • Free-form text content (FREE mode)
  • Visual customization (colors, modules, logo)
  • Size and layout constraints
Parameters
availableHeightForQrCodethe vertical space (in pixels) available for rendering the QR code in the UI; used to calculate optimal display size
currentModethe encoding mode: Mode#MECARD for structured contact data or Mode#FREE for arbitrary text content
freefree-form text content for non-MECARD encoding; used when currentMode is Mode#FREE; can be any string (URL, message, JSON, etc.)
namefull name of the contact (MECARD mode); typically in "LastName, FirstName" format; ignored in FREE mode
orgorganization or company name associated with the contact (MECARD mode); represents the business or employer; ignored in FREE mode
teltelephone number in international or local format (MECARD mode); should follow E.164 format when possible (e.g., "+33123456789"); ignored in FREE mode
emailemail address of the contact (MECARD mode); must be a valid email format (e.g., "user@example.com"); ignored in FREE mode
adrpostal address of the contact (MECARD mode); can include street, city, postal code, and country; typically comma or newline-separated; ignored in FREE mode
urlwebsite or profile URL (MECARD mode); can be a personal site, LinkedIn profile, portfolio, or social media link; must be a valid URL format; ignored in FREE mode
logoPathfile system path to an optional logo image to embed in the QR code center; supports PNG, JPG, JPEG formats; if null or empty, no logo is embedded; logo size is automatically scaled based on ratio
sizethe target dimension (width and height) of the generated QR code image in pixels;
marginthe outer margin (quiet zone) around the QR code in modules (QR matrix units); valid range: 0-10; values below 4 may cause scanning issues on some readers; recommended: 3-4 for reliable detection
ratiothe logo visibility ratio as a decimal (0.0-1.0); determines the proportion of the QR code center occupied by the logo; typical range: 0.2-0.3; values above 0.3 may compromise scannability despite error correction
qrColorthe color of the QR code modules (dark pixels); must provide sufficient contrast with bgColor; default: Color#BLACK; custom colors allowed but high contrast is essential for reliable scanning
bgColorthe background color (light pixels) of the QR code; must provide sufficient contrast with qrColor; default: Color#WHITE; custom colors allowed but high contrast is essential for reliable scanning
isRoundedModuleswhether to render QR code modules with rounded corners instead of square pixels; true for aesthetic rounded style, false for traditional square modules; rounded modules may reduce scannability on some readers but are generally supported with high error correction
